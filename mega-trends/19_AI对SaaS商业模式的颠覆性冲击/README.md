# 议题19：AI对SaaS商业模式的颠覆性冲击

## 核心判断

市场普遍认为AI是SaaS的增量收入来源（"AI溢价"），但实际上AI是SaaS的利润杀手。AI功能同质化导致定价权丧失，SaaS毛利率从80%向60-70%压缩。Copilot/AI助手正在改变软件定价从按席位（seat-based）转向按用量（usage-based），这本质上是收入模式降级——用户用得更少时付得更少，而不是按人头固定收费。同时，AI编程工具（Cursor/GitHub Copilot/Claude Code）正在让企业自建软件替代SaaS订阅成为可能，特别是中端市场。Salesforce/ServiceNow/HubSpot面临结构性估值压缩。软件是过去20年最好的商业模式，但AI正在系统性地侵蚀其护城河和定价权。属反向共识级议题。

**📊 最新评级：[通过]**（2026.06.05从「有条件通过」升级）
- $2万亿SaaS市值蒸发，P/E 22.7x首低于标普500（2026年2月SaaSpocalypse确认）
- AI原生产品毛利率仅52-70%（传统SaaS 78-85%），20个百分点永久性压缩
- Salesforce Agentforce $1.2B ARR(+205%)证明龙头可转型，但水平SaaS(HUBS/TEAM/Figma)已跌70-80%
- Klarna企业级AI替代SaaS案例标志着威胁从理论进入企业决策
- 策略：做空水平/中端SaaS + 做多AI原生/基础设施

## 反直觉视角

**软件是过去20年最好的商业模式，但AI正在同时侵蚀护城河和定价权。** SaaS的"魔力"在于：高切换成本锁定客户（护城河）+ 按席位收费的经常性收入（定价权）+ 80%毛利率（运营杠杆）。AI正在系统性地瓦解这三个支柱：（1）AI编程让自建替代方案的成本降低3-5倍，切换成本下降；（2）AI功能同质化，每家SaaS都在加AI，但没人能独占定价权；（3）AI推理成本计入COGS，毛利率被侵蚀。这不是SaaS公司在"转型AI"，而是AI在结构性降低SaaS商业模式的盈利能力。

## 受冲击的SaaS商业模式支柱

```
SaaS商业模式 vs AI冲击
├── 护城河（切换成本）
│   ├── 传统：数据锁定+工作流嵌入+员工培训成本
│   ├── AI冲击：AI编程降低自建成本，API标准化降低切换成本
│   └── 评估：中端市场护城河显著减弱，大型企业仍有数据锁定
│
├── 定价权（seat-based定价）
│   ├── 传统：按人头收费，用户增长=收入增长
│   ├── AI冲击：AI按用量收费，用得少付得少
│   └── 评估：从"订阅税"变"使用费"，收入可预测性下降
│
├── 毛利率（80%+）
│   ├── 传统：软件边际成本接近零
│   ├── AI冲击：每次AI调用都有推理成本（COGS增加）
│   └── 评估：AI密集型SaaS毛利率可能降至60-70%
│
└── 增长飞轮（land-and-expand）
    ├── 传统：免费/低价获客→增值升级→席位扩张
    ├── AI冲击：AI功能让免费版"够用"，升级动力减弱
    └── 评估：upsell路径被压缩
```

## 关注模型

- SaaS公司AI功能定价 vs AI推理成本（毛利率压缩）
- Seat-based收入占比变化（定价模式转型进度）
- 企业SaaS流失率趋势（客户留存压力）
- AI编程工具采用率（自建替代可行性）
- SaaS公司估值倍数变化（EV/Revenue）
- AI原生软件公司对传统SaaS的市场份额侵蚀
- 企业IT预算中SaaS vs 自建的分配变化
- SaaS公司研发支出中AI投入占比（是否挤压利润）
