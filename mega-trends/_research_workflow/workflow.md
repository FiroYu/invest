# Agent 执行手册：7 轮投研分析流程

> 本文件是 Agent 执行投研分析的完整操作指南。
> 每个 `Round` 是一个独立阶段，包含输入、动作、输出。
> 角色的具体 prompt 见 `role_prompts.md`，输出格式见 `output_format.md`。

---

## 元信息

- **适用范围**: mega-trends 目录下所有编号课题（01-21）
- **执行频率**: 每个课题每轮全量更新时执行一次
- **并发策略**: Round 1 + Round 2 可并行；Round 3-6 严格串行

---

## Round 0: 准备 + 假设框定

**执行者**: 投资总监（CIO / orchestrator）

### 输入

- 课题编号 N
- 课题目录 `mega-trends/N_xxx/` 下全部文件

### 步骤

1. **读取历史文件**
   - `README.md` — 核心判断、辩论评级、关键指标
   - `timeline.md` — 关键事件时间线
   - `technical_analysis.md` — 技术深度分析
   - `economic_impact.md` — 投资影响分析
   - `updates/` 下所有文件（按时间倒序，最近 3 个必读）

2. **读取横向风险因子**
   - `mega-trends/00_横向风险因子矩阵.md`
   - 标注该课题对 4 个风险因子的敏感度

3. **读取关联课题**
   - 查 README 索引表，找到与本课题共享横向风险因子的课题
   - 读取这些课题的 `README.md`（仅核心判断部分）

4. **构建假设清单**

   对该课题当前投资论点，列出以下内容：

   ```
   ## 核心假设（论点成立必须满足）
   H1: [假设] — 验证方式: [如何判断成立/不成立]
   H2: [假设] — 验证方式: [...]
   H3: [假设] — 验证方式: [...]
   （3-5 个）

   ## 退场标准（出现即放弃或降级）
   K1: [触发条件] → 后果: [降级为技术关注 / 移除 / 反转做空]
   K2: [触发条件] → 后果: [...]
   （2-3 个）

   ## 与上次分析的 Delta
   D1: [变化内容] — 影响: 正/负/中性
   D2: [变化内容] — 影响: [...]
   （列出与上一轮更新之间的关键变化）
   ```

5. **打包背景包**

   将以上所有内容整合为一个背景包文档，格式如下：
   ```
   # 课题 #N 背景包

   ## 一、历史摘要
   [从 README/economic_impact/timeline 提炼的核心结论]

   ## 二、横向风险因子
   [该课题对 4 个因子的敏感度 + 当前因子状态]

   ## 三、关联课题
   [相关课题编号 + 核心判断]

   ## 四、假设清单
   [H1-H5 + 验证方式]

   ## 五、退场标准
   [K1-K3]

   ## 六、Delta
   [D1-Dn]
   ```

### 输出

- 背景包文档 → 传递给 Round 1 的三个研究员和 Round 2 的魔鬼辩护人

## Round 1: 三研究员独立调研

**执行者**: 基本面研究员 A / 催化剂研究员 B / 宏观调研员 C（**并行执行**）

**关键约束**: 三人**互不可见**，保证独立性。

### 共同输入

- Round 0 输出的背景包

### 各角色任务

#### 研究员 A（`fundamental`）— 基本面与估值

搜索并分析：
- 最新季度财报（收入、利润、 margins、guidance 变化）
- 估值指标（EV/EBITDA、P/E、Forward P/E、PEG）
- 行业对标和竞争格局变化
- 市场份额变动
- 盈利增速和预期修正

#### 研究员 B（`catalyst`）— 催化剂与管线

搜索并分析：
- 近 30-90 天的重大事件（产品发布、审批、政策变化）
- 管线进展 / 产品路线图更新
- 竞品动态（新进入者、定价变化）
- M&A 传闻或确认
- 监管环境变化

#### 研究员 C（`macro`）— 宏观与地缘

搜索并分析：
- 利率/货币政策对该课题的传导
- 地缘冲突升级/缓和的影响
- 供应链数据（运费、库存、订单）
- 汇率和大宗商品影响
- 政策/财政变化

### 各角色必须包含的内容

每份报告都必须回应假设清单中的每个假设，并提供情景分析：

```
## 假设验证
| 假设 | 判断 | 证据 | 置信度(1-5) |
|------|------|------|:-----------:|
| H1   | 成立/不成立/待验证 | ... | 4 |

## 情景分析
| 情景 | 概率 | 核心驱动 | 预期回报 |
|------|:----:|---------|---------|
| Bull | % | ... | +XX% |
| Base | % | ... | +XX% |
| Bear | % | ... | -XX% |

## 时间维度
- 短期(1-3月): ...
- 中期(3-12月): ...
- 长期(1-3年): ...
```

### 输出

- 三份独立研究报告 → 传递给 Round 2（魔鬼辩护人此时已完成）和 Round 3（PM）

## Round 2: 魔鬼辩护人

**执行者**: 魔鬼辩护人（`devil`）

**与 Round 1 并行启动**，只看背景包，不看研究员报告。

### 输入

- Round 0 输出的背景包（**不含** Round 1 的研究员报告）

### 步骤

1. 对每个核心假设（H1-H5），搜索**证伪证据**
2. 搜索做空报告、卖方 bear case、行业批评、失败案例
3. 构建完整反方论据链
4. 提出"最可能的失败模式" Top 3
5. 寻找历史类比（类似技术/行业最终失败的案例）

### 输出格式

```
## 反方论据
R1: [反方论点] — 证据强度: 强/中/弱 — 来源: [...]
R2: [...]

## 最可能的失败模式（Top 3）
1. [失败模式] — 触发条件: [...] — 概率: X%
2. [...]

## 历史类比
- [类似案例] 最终 [成功/失败] 的原因: ...

## 对退场标准的补充
- 建议新增退场标准: [...] → 后果: [...]
```

### 输出

- 魔鬼辩护人报告 → 传递给 Round 3（PM）

## Round 3: 基金经理审阅 + 跨课题传导

**执行者**: 基金经理（`pm`）

### 输入

- 三份研究员报告（Round 1）
- 魔鬼辩护人报告（Round 2）
- 背景包（Round 0）

### 步骤

#### Part A: 质疑与挑战

1. 对比三份研究员报告的**分歧点**
2. 对比研究员共识 vs 魔鬼辩护人的**对立点**
3. 对每个研究员的核心假设提出**至少 3 个质疑**
4. 识别三份报告的**共同盲区**（三人都没提到但重要的维度）
5. 给出**补强清单**（哪些数据/论据最需要补充）

#### Part B: 跨课题传导分析

1. 从背景包中提取关联课题列表
2. 分析本课题结论对关联课题的传导效应
3. 识别组合层面的共同风险暴露

### 输出格式

```
## 质疑清单
| # | 质疑对象 | 质疑内容 | 要求补强的数据 |
|---|---------|---------|-------------|
| Q1 | 研究员A | ... | ... |
| Q2 | 研究员B | ... | ... |

## 分歧点汇总
| 分歧 | A的观点 | B的观点 | C的观点 | 魔鬼辩护人 | 我的初步判断 |
|------|---------|---------|---------|-----------|-----------|

## 盲区清单
- [三份报告都没覆盖的重要维度]

## 补强要求（按优先级）
1. [最需要补充的数据/论据]
2. [...]

## 跨课题传导
| 关联课题 | 传导路径 | 方向 | 强度 |
|---------|---------|:----:|:----:|
| #XX | 本课题→#XX: [...] | 正/负 | 强/中/弱 |

## 组合风险
- 共同风险因子 [X] 触发时，以下课题同时受损: #XX, #XX, #XX
```

### 输出

- PM 审阅备忘录 → 传递给 Round 4 的研究员

## Round 4: 深度研究 + 交叉评审（多轮收敛）

**执行者**: 三研究员 + 交叉评审席

### Phase 4a: 回应质疑（三研究员各自修订）

**输入**:
- 各自的 Round 1 报告
- PM 的质疑清单和补强要求
- 魔鬼辩护人的反方论据

**动作**:
- 每个 researcher 针对 PM 的质疑和魔鬼辩护人的挑战，补充搜索和数据
- 修订自己的报告（标注修订部分）
- 更新假设验证、情景分析、时间维度

### Phase 4b: 交叉评审

**规则**:
- A 评 B 和 C 的修订报告
- B 评 A 和 C 的修订报告
- C 评 A 和 B 的修订报告

**评审加权**:
- 同领域评审（如基本面评基本面）权重 60%
- 跨领域评审权重 40%
- 领域匹配: fundamental ↔ fundamental, catalyst ↔ catalyst, macro ↔ macro

**评审维度**:

| 维度 | 分值 | 含义 |
|------|:----:|------|
| 论据充分性 | 1-5 | 数据/证据是否支撑结论 |
| 逻辑一致性 | 1-5 | 内部推理是否自洽 |
| 信息新颖度 | 1-5 | 是否发现上次没有的新信息 |
| 反方回应质量 | 1-5 | 是否有效回应魔鬼辩护人的挑战 |
| 置信度 | 1-5 | 对自己评分的确信程度 |

**投票**:
每人给出方向性投票：看多 / 中性 / 看空

### Phase 4c: 收敛判定

**判定规则**:
- 三人方向一致（全看多或全看空）→ **收敛，进入 Round 5**
- 两人一致 + 一人分歧 → 记录分歧原因，进入 Round 5，由 CIO 裁定
- 三人全部分歧 → **启动补充轮**（聚焦分歧点再搜索一轮，最多 1 轮补充）

### 输出

- 三份修订报告
- 三份交叉评审意见 + 评分 + 投票
- 收敛状态说明

## Round 5: 投资总监综合

**执行者**: 投资总监（CIO）

### 输入

- 背景包（Round 0）
- 三份修订报告 + 交叉评审 + 投票（Round 4）
- 魔鬼辩护人报告（Round 2）
- PM 审阅备忘录（Round 3）

### 步骤

1. **综合所有视角**，提炼共识与分歧
2. **对分歧做裁定**（采信哪方 + 理由）
3. **假设验证总结**（表格汇总所有角色的判断）
4. **情景加权计算**（bull/base/bear × 概率 = 加权期望回报）
5. **更新辩论评级**
6. **给出操作建议**
7. **明确退场标准**
8. **设定下次复查触发条件**

### 输出格式

见 `output_format.md` 中 `_synthesis.md` 模板。

### 输出

- `_synthesis.md` 最终综合报告

---

## Round 6: 写入仓库 + 增量追踪

**执行者**: CIO（orchestrator）

### 步骤

1. 在课题的 `updates/` 下创建日期目录
2. 写入所有文件（`_synthesis.md` + 支撑材料）
3. 更新课题 `README.md` 的核心判断、辩论评级
4. 如果横向风险因子有变化，更新 `00_横向风险因子矩阵.md`
5. 如果关联课题受传导影响，标记待更新

### 文件结构

```
updates/
  YYYY.MM.DD_round_N/
    _synthesis.md                 ← 必读：CIO 综合报告
    researcher_A_fundamental.md   ← 支撑材料
    researcher_B_catalyst.md
    researcher_C_macro.md
    devil_advocate.md
    pm_review.md
    peer_reviews.md
```

### `_synthesis.md` 必须包含的 Delta 摘要

```
## 与上次的 Delta
| 维度 | 上次 | 本次 | 变化 |
|------|------|------|------|
| 辩论评级 | [旧] | [新] | [原因] |
| 核心假设 H1 | [旧状态] | [新状态] | [原因] |
| 新增风险 | — | [X] | [说明] |
| 新增机会 | — | [X] | [说明] |
```

## 并发与依赖关系

```
Round 0 ──→ Round 1 (A, B, C 并行) ──→ Round 3 (PM) ──→ Round 4 ──→ Round 5 ──→ Round 6
          ──→ Round 2 (魔鬼辩护人, 与 Round 1 并行) ──↗
```

- Round 0 必须先完成
- Round 1 和 Round 2 可以并行
- Round 3 等待 Round 1 + Round 2 都完成
- Round 4 等待 Round 3
- Round 5 等待 Round 4
- Round 6 等待 Round 5

## Token 预算

每个课题单次完整分析预估消耗：
- Round 0: ~5K tokens
- Round 1: ~15K × 3 = ~45K tokens
- Round 2: ~10K tokens
- Round 3: ~10K tokens
- Round 4: ~15K tokens（含交叉评审）
- Round 5: ~10K tokens
- Round 6: ~5K tokens
- **合计: ~100K tokens / 课题**
