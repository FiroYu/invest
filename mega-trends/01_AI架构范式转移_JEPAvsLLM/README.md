# AI架构范式转移：JEPA vs LLM

## 一句话判断
> 杨立昆的 JEPA 架构在视觉理解和机器人规划领域已验证有效性（1B参数达到T级LLM的部分能力），但在纯语言生成上尚未证明自己。长期来看JEPA可能降低算力需求10-100倍，但短期内（1-3年）LLM推理需求仍在爆发式增长（token使用量年增250%+），AI CapEx超级周期尚未见顶。需要同时跟踪"训练侧架构竞争"和"推理侧规模战争"两条线。

## 推理vs训练：两条投资主线

| 维度 | 训练侧（当前主导） | 推理侧（加速崛起） |
|------|-------------------|-------------------|
| 核心产品 | 通用GPU (NVIDIA H100/B200) | 定制ASIC (Broadcom/Google TPU) |
| 市场 | AI ASIC $120亿→$300亿(2027) | — |
| 经济模型 | 一次性CapEx | 持续运营成本 |
| 决定因素 | 模型规模+参数量 | 用户量×调用频率×token单价 |
| 关键瓶颈 | 先进制程(GAA)、HBM | 先进封装(CoWoS)、互联带宽 |

**重要**：推理经济决定AI应用能否实现商业回报。如果token单价不降到$0.01/1M以下，多数AI应用可能无法盈利。详见议题#23。

## AI颠覆风险视角

Blackstone要求所有投资委员会备忘录前两页必须包含"AI风险分析"。以下行业面临"出租车牌照级"结构性颠覆风险：
- 外包/BPO（AI Agent替代，详见#10）
- 传统CRO（AI药物发现替代，详见#24）
- 汽车保险（自动驾驶替代，详见#12）
- 中等市场银行贷款（私募信贷替代，详见#22）

## 为什么重要

- **人类影响**：AI从"数据中心专属"变为"边缘设备可运行"，智能将无处不在（手机、汽车、机器人）
- **经济影响**：当前万亿美元级别的AI基础设施投资（GPU集群、数据中心、核电站）可能面临结构性过剩
- **时间敏感度**：LeCun 预测 3-5 年内 JEPA 取代 LLM（2025年MIT演讲）。V-JEPA 2 已于2025年发布，VL-JEPA论文已上线，技术验证正在加速

## 核心争议点

### 正方（LeCun / Meta AI）
- 自回归 LLM 无法真正理解世界、缺乏规划和推理，单纯 scaling 无法通向 AGI
- JEPA 在潜空间预测，参数效率远高于 token 级生成
- 人类大脑 4 岁幼儿以几瓦功耗学到的世界知识远超最大 LLM 的训练数据

### 反方（OpenAI / Anthropic / Scaling 派）
- Scaling law 尚未触顶，更大模型 + 更多数据仍在持续提升能力
- LLM 已形成强大的生态壁垒（开发者工具、应用、用户习惯）
- JEPA 在语言生成等核心商业场景上缺乏有力证据
- 历史上"更高效的架构"（如RNN、Capsule Net）并不总能取代"粗暴但有效"的方案

### 当前共识
- JEPA 在视觉理解和物理推理上确实优于同等规模的生成式模型
- 纯语言生成领域，LLM 仍是不可替代的
- 两者可能长期共存：LLM 负责语言，JEPA 负责世界模型

## 关键指标（判断验证用）

| 指标 | 当前值（2026.05） | 判断阈值 | 数据来源 |
