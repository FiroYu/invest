# 议题模板

创建新议题时，将本文件夹复制为 `NN_议题简称/`，然后填写以下文件。

## README.md — 议题概述

```markdown
# [议题标题]

## 一句话判断
> [用一句话概括这个议题的核心论点和你当前的最佳判断]

## 为什么重要
- [对人类生活的影响]
- [对经济结构的影响]
- [时间敏感度：什么时候会成为现实？]

## 核心争议点
- 正方观点：
- 反方观点：
- 当前共识（如有）：

## 关键指标（判断验证用）
| 指标 | 当前值 | 判断阈值 | 数据来源 |
|------|--------|----------|----------|
| [可量化的指标1] | | | |
| [可量化的指标2] | | | |

## 文件索引
- [timeline.md](timeline.md) — 关键事件时间线
- [technical_analysis.md](technical_analysis.md) — 技术深度分析
- [economic_impact.md](economic_impact.md) — 经济影响与投资分析
- [updates/](updates/) — 持续更新日志
```

## timeline.md — 时间线格式

```markdown
# [议题标题] — 事件时间线

## 2026
- **YYYY-MM-DD**: [事件描述] — [来源链接](url)
```

## economic_impact.md — 投资分析格式

```markdown
# [议题标题] — 经济影响与投资分析

## 受益行业
| 行业 | 逻辑 | 受益程度 | 代表标的 |
|------|------|----------|----------|
| | | 高/中/低 | |

## 受损行业
| 行业 | 逻辑 | 受损程度 | 代表标的 |
|------|------|----------|----------|
| | | 高/中/低 | |

## 基础设施变化
- 芯片：
- 存储：
- 电力：
- 网络：

## 投资策略
### 短期（0-6个月）
### 中期（6-24个月）
### 长期（2年以上）
```

## updates/ — 更新日志格式

文件命名：`YYYY.MM.DD_简述.md`

```markdown
# YYYY.MM.DD 更新

## 新进展
- [描述]

## 判断更新
- [是否需要修正之前的判断]

## 待关注
- [下一步需要追踪的信息]
```
